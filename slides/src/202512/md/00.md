## Is this course for you?

- You're starting with a new code and don't want to deal with writing the I/O layer
- Your I/Os take too long and you wish to dump images, or extracts as simulations run
- You'd like to connect `live` to a running simulation and inspect all variables at full temporal and spatial resolution
- You'd like to switch at the click of a button between,
    - A) writing data to disk, and
    - B) sending your data to another `in-memory` consumer

[slides](https://jfavre.github.io/InSitu-Vis-Tutorial/slides)    : https://jfavre.github.io/InSitu-Vis-Tutorial/slides

## Technical setup: Catalyst

[**A ParaView uenv**](https://docs.cscs.ch/software/sciviz/paraview/#one-time-setup)
```
    - uenv image pull build::paraview/6.0.1:v1
    - uenv image run paraview/6.0.1:v1 -- cp -r /user-environment/paraview/helpers/. ~/bin
```

- uenv start paraview/6.0.1:v1
- pvpython interactive application
- ParaView parallel servers
    - pvbatch,
    - pvserver
- ParaView Catalyst
- Python scripting

## Technical setup: Ascent

**An Ascent uenv**
```
uenv image add  ascent/0.9.5:v1@daint%gh200  /capstor/store/cscs/cscs/public/uenvs/ascent/20251211-gh200.python.squashfs
```
- (only once). Then, do a standard `uenv start`<br>
- uenv start ascent/0.9.5:v1 --view=spack<br>
- export SPACK_SYSTEM_CONFIG_PATH="/user-tools/config"<br>
- no client application<br>
- python -c "import ascent"
